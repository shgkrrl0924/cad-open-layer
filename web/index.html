§01 / CAPABILITIES

Three primitives.
Round-trip preserving.

001 Parse Streaming DXF reader (R12 → R2018). 12 entity types, BLOCKS, HEADER vars. Zero-copy where possible. 23k entities / 178ms measured on multifamily floor plans. cad_dxf_parser
002 Extract Five algorithms surface architectural semantics: walls (parallel-line-pair), openings (block projection), rooms (DCEL faces), dimensions (chain reconstruction), grid axes. cad_extract
003 Synthesize Floorplan → DXF round-trip. Wall counts, opening counts, room counts, grid axes preserved exactly across parse → write → reparse cycles. Layers retained per-wall. cad_synthesize
§02 / RUNTIMES

Rust core.
Browser, Node, MCP bindings.

004 Rust Add cad-open-layer = "0.1" to your Cargo.toml. Eight-crate workspace, zero unsafe, MSRV 1.85. cargo add
005 Browser WASM bindings expose parseDxfSummary() and extractFloorplanFromDxf() as plain JS functions. Try the Playground → cad_wasm
006 MCP server JSON-RPC over stdio. Drop into Claude Desktop or Cursor as an AI-agent tool. Two methods: parse_dxf, extract_floorplan. cad_mcp
§03 / WHY

Three existing options.
Each one compromises.

[ APS ]
Autodesk Platform Services works, but vendor lock-in and recurring cost. You don't own your access path.
[ OSS ]
Existing open-source libraries read DXF but break round-trip. Semantic information (which lines are walls, which arcs are doors) is lost.
[ COM ]
AutoCAD COM/ActiveX automation works only on Windows, only with a licensed AutoCAD running, and is too slow for server pipelines.
[ THIS ]
CAD Open Layer is the fourth path: open source, programmable, vendor-neutral, with semantic preservation as a first-class goal. Built for AI agents, BIM SaaS, and SI pipelines that don't want a person clicking through AutoCAD on their critical path.
